# 产品宪法 · Agentic Investment Management System (AIMS)

## 一句话定位

**不是 AI 炒股工具，而是能逐渐吸收你投资方法论的「模拟基金经理系统」。**

## 核心目标（非预测涨跌）

让 AI 按你的投资框架，持续完成闭环：

```
信息收集 → 研究分析 → 形成观点 → 构建组合 → 模拟交易 → 记录结果 → 复盘归因 → 迭代策略
```

## 硬性原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | 模拟优先 | 不做实盘自动交易；实盘接口 Phase 4+ 再考虑 |
| 2 | 观点 ≠ 交易 | 研究看好 ≠ 自动买入；必须经过组合约束与风控 |
| 3 | 决策可追溯 | 每次动作保留理由、假设、参考信息、复盘条件 |
| 4 | 结构化输出 | AI 输出尽量 JSON Schema 落库，禁止纯自然语言决策 |
| 5 | 记忆可进化 | 错误、用户反馈、归因结果沉淀为 Decision Memory |
| 6 | MVP 分期 | 先组合+决策日志，再信息层，再多 Agent，最后进化 |

## Agent 角色（8 个）

| Agent | 职责 |
|-------|------|
| Data Agent | 多源数据接入、清洗、更新调度 |
| Research Agent | 公司/行业研究、财报解读、事件跟踪 |
| Valuation Agent | DCF/PE/PB/SOTP、情景与目标价 |
| Factor Agent | 因子、风格暴露、量化信号 |
| Risk Agent | 集中度、回撤、流动性、估值与事件风险 |
| Portfolio Agent | 资产配置、权重、调仓草案 |
| CIO Agent | 汇总观点，输出最终模拟决策单 |
| Review Agent | 复盘、归因、教训提炼、规则更新建议 |

## 用户投资风格（可配置，系统逐步学习）

- 主观多头、基本面驱动
- 重视产业趋势与估值安全边际
- 主战场：A 股 + 港股
- 禁止项、偏好项由用户反馈库维护

## Phase 边界

- **Phase 1**：股票池、模拟组合、交易、净值、PnL、日报、决策日志
- **Phase 2**：新闻/公告/财报/研报解析、研究页
- **Phase 3**：多 Agent 协作与调仓建议
- **Phase 4**：回测、归因、记忆库、策略进化
