# 核心页面设计（第一版 7 页）

技术：**Next.js** + 路由分组 `(dashboard)`。所有 AI 生成内容展示「结构化卡片 + 原始 JSON 调试开关」。

---

## 1. Dashboard 总览 `/`

**目标**：30 秒掌握组合状态与待办。

| 区块 | 内容 |
|------|------|
| KPI 卡片 | 净值、日涨跌、累计收益、最大回撤、现金占比 |
| 净值曲线 | 组合 vs 基准（可切换 1M/3M/YTD） |
| 今日待办 | 待审批决策、待复盘决策、新结构化事件数 |
| 行业暴露 | 条形图 Top 5 |
| 最新决策 | 最近 3 条，链到 Decisions 详情 |
| 记忆摘录 | 1-2 条活跃 `memory_entries`（Phase 4，MVP 可占位） |

---

## 2. Portfolio 模拟组合 `/portfolio`

| 区块 | 内容 |
|------|------|
| 持仓表 | 标的、权重、成本、现价、浮盈亏、行业 |
| 暴露分析 | 行业 / 市场（A/H）饼图 |
| 交易历史 | 筛选：日期、标的、来源（manual/agent） |
| 操作 | 手动模拟交易抽屉；「生成调仓建议」按钮（Phase 3） |
| 风险仪表 | 距上限：单票、行业、现金下限（Risk 规则结果） |

**交互**：点击持仓 → 跳转 Research 该公司页。

---

## 3. Watchlist 股票池 `/watchlist`

| 功能 | 说明 |
|------|------|
| 多列表 | 核心池 / 跟踪池 / 想法池 |
| 列 | 代码、名称、评级（研究观点）、目标价区间、上次决策、备注 |
| 操作 | 加入/移除、写 thesis_summary、触发「更新研究」（Phase 2） |
| 批量 | 导出 CSV；选中后「发起观察决策」 |

---

## 4. Research 公司研究 `/research/[symbol]`

**研究观点与交易分离**：页面上方 **Research View**，下方 **相关 Decisions**。

| Tab | 内容 |
|-----|------|
| 概览 | 十段式基本面卡片（商业模式…投资结论） |
| 估值 | 情景表 + 模型参数（DCF/PE 等切换） |
| 因子 | Phase 3：动量/价值/质量等雷达图 |
| 事件时间线 | structured_events 过滤该公司 |
| 文档 | 上传研报/纪要，解析状态 |
| 历史观点 | research_views 版本 diff |

**操作**：用户可编辑/批注；「生成研究草稿」→ Research Agent。

---

## 5. News & Events 信息流 `/events`

| 功能 | 说明 |
|------|------|
| 过滤器 | 市场、行业、事件类型、影响方向、时间敏感度 |
| 卡片 | 结构化字段：公司、事件类型、影响维度、置信度、follow_ups |
| 操作 | 标记已读；「加入跟踪」；「生成事件点评」（Phase 2） |
| 详情侧栏 | 原文 + 提取 JSON |

---

## 6. Decisions 决策日志 `/decisions` ⭐ 最重要

**系统护城河页面。**

### 列表

| 列 | 说明 |
|----|------|
| 日期 | |
| 标的 | |
| 动作 | buy/sell/add/reduce/hold/watch/ban |
| 仓位变化 | 5% → 7% |
| 信心 | B+ |
| 状态 | draft/approved/executed |
| 结果 | Phase 4：收益率标签 |

筛选：组合、状态、动作、Agent/人工。

### 详情页 `/decisions/[id]`

```
┌─────────────────────────────────────────┐
│ 决策单头：动作、仓位、信心、持有周期      │
├─────────────────────────────────────────┤
│ 决策理由（自然语言 + 结构化 cio_summary）│
├─────────────────────────────────────────┤
│ 核心假设列表（可勾选后续验证状态）        │
├─────────────────────────────────────────┤
│ 主要风险 | 复盘/止损条件（假设驱动）      │
├─────────────────────────────────────────┤
│ 参考信息时间线（news/filing/valuation）   │
├─────────────────────────────────────────┤
│ Agent 轨迹折叠面板（Phase 3）            │
├─────────────────────────────────────────┤
│ 用户反馈区：评分、纠正、标签              │
├─────────────────────────────────────────┤
│ 执行记录：关联 trades                    │
├─────────────────────────────────────────┤
│ 结果复盘（Phase 4）：对错、教训链接       │
└─────────────────────────────────────────┘
```

**操作**：Approve → Execute 模拟成交；Supersede 新决策。

## 7. Review 复盘归因 `/review`

| 区块 | Phase |
|------|-------|
| 开放决策看板 | 到达复盘日但未 closed 的 decisions |
| 周期复盘 | 周/月：净值归因、行业贡献（Phase 4） |
| 决策复盘列表 | what_went_right/wrong |
| 记忆库浏览 | memory_entries 分类：lesson / anti_pattern / rule |
| 规则建议 | Review Agent 产出待用户确认条目 |

---

## 全局 UX 约定

1. **研究观点** 用蓝色标签，**交易动作** 用橙色标签，避免混淆。
2. 任何 Agent 按钮显示「预计输入/输出 schema」。
3. 空状态引导：无决策时提示「从 Watchlist 创建第一条观察决策」。
4. 移动端：MVP 仅需 Dashboard + Decisions 只读适配。
